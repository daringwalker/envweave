# 变更记录

本项目遵循语义化版本。正式稳定版发布前，仓库格式和交互仍可能发生不兼容变化。

## 0.1.0-alpha.4 - 2026-07-16

### 新增

- 目录配置支持 `replace`、`merge`、`keep-existing` 三种恢复策略，以及安全相对排除路径。
- 恢复预检展示逐路径新增、覆盖、保留和删除影响；存在删除时强制人工确认。
- Git 同步页支持识别 rebase 冲突、列出冲突文件、安全继续或中止变基。
- 智能扫描选择配置后可在右侧自动预览文本内容。

### 改进

- 统一浅色、深色与 Monaco 编辑器配色，并加入填充业务数据的主题验收页面。
- 新添加的目录默认采用保留本机独有内容的 `merge` 策略。
- 恢复计划同时绑定仓库和本机目标状态，预检后任一侧变化都必须重新确认。
- 软件包扫描结果和便携应用来源信息在页面切换及重新扫描后保持稳定。

### 修复

- 修复恢复缺失的嵌套目标时，父路径不是目录可能导致事务回滚失败的问题。
- 修复 Git 推送错误显示为 `[object Object]`，并为缺失远程地址提供可操作提示。

## 0.1.0-alpha.3 - 2026-07-15

### 修复

- AppImage 在 Arch 系发行版启动时预加载系统 Wayland 客户端库，修复 Ubuntu 构建产物与 Arch EGL 栈冲突导致的空白窗口。
- Linux 自动化新增 Ubuntu 构建 AppImage 到 Arch 运行的跨发行版 GUI 验收。

## 0.1.0-alpha.2 - 2026-07-15

### 修复

- Linux Wayland 会话默认禁用 WebKitGTK DMABUF 快速渲染路径，修复部分 Arch KDE Wayland 环境启动 AppImage 后界面空白的问题。
- Linux 虚拟机验收现在直接启动最终 AppImage，避免只测试打包前原生二进制而遗漏 AppImage 运行时问题。

## 0.1.0-alpha.1 - 2026-07-15

首个可安装的开发者预览版。

### 已实现

- Linux/macOS 桌面 GUI、仓库首次使用流程和 Git 同步。
- 配置知识库、智能扫描、批量添加和 Monaco 可视化差异编辑。
- 用户级文件的事务恢复、写前备份、失败整批回滚和中断事务恢复。
- pacman/AUR、Homebrew、Mac App Store 和 Flatpak 软件清单。
- Linux Desktop Entry 扫描，以及 AppImage/解压运行程序的来源记录。
- Ubuntu Deb/AppImage、Arch AppImage 和 macOS App/DMG 构建基础。

### 已知限制

- 目录恢复尚未提供逐文件删除预览和完整合并策略。
- 系统级配置只支持采集、查看和比较，不直接写回。
- 不提供秘密文件加密存储；敏感内容默认需要额外确认。
- macOS 公共下载版本必须完成 Developer ID 签名和 Apple 公证。
- 软件包与配置阶段尚未形成可暂停、可续跑的单一持久任务。
