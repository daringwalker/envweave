# 安全策略

## 支持范围

`0.1.0-alpha.1` 是开发者预览版，只接收当前最新 Alpha 版本的安全修复。

## 报告漏洞

请使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口，不要在公开 Issue 中提交利用细节、用户配置、仓库地址、访问令牌或日志中的隐私信息。

报告应尽量包含受影响版本、平台、复现步骤、潜在影响以及经过脱敏的日志。收到报告后，维护者应先确认影响范围，再协调修复与披露时间。

## 用户数据边界

- EnvWeave 默认在本地处理配置内容，不应将秘密文件提交到未加密 Git 仓库。
- 在恢复前保留独立于 EnvWeave 仓库的系统备份。
- 只从可信发布页获取安装包，并按发布文档校验 SHA-256。
