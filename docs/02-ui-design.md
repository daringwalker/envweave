# 界面与交互设计

## 1. 信息架构

应用使用单窗口、左侧导航结构：

- **概览**：状态摘要、主要动作、最近活动。
- **配置文件**：搜索、筛选、批量操作、项目详情。
- **对比工作台**：本机与仓库版本的可视化对比、编辑和逐块采用。
- **软件包**：扫描、筛选、保存清单、比较与恢复软件。
- **同步**：远程状态、提交预览、拉取与推送。
- **备份**：按操作分组的备份与恢复。
- **设置**：仓库、Git、外观和诊断。

首次启动时不显示完整主界面，而是进入三步引导。

## 3. 首次启动引导

### 步骤一：选择来源

以三个大卡片呈现：

- 新建配置仓库（推荐给首次使用者）
- 克隆 Git 远程仓库
- 打开本地仓库

### 步骤二：连接与路径

- 新建：仓库保存位置、可选远程 URL。
- 克隆：远程 URL、本地保存位置、连接测试。
- 打开：系统目录选择器、清单格式检查。

输入框旁直接显示验证结果，错误不使用只有“失败”字样的弹窗。

### 步骤三：选择配置

展示常见配置扫描建议，例如 Shell、Git、SSH 客户端配置、编辑器。敏感项默认不勾选。完成前显示仓库路径和预计文件数量。

## 4. 对比编辑工作台

双击有改动的配置，或点击“比较并编辑”，进入占满内容区的工作台：

```text
┌────────────────────────────────────────────────────────────────────────────┐
│  ‹ 返回配置   Zsh · ~/.zshrc                 已保存   3 处差异   ↑ 1/3 ↓ │
├────────────────────────────────────────────────────────────────────────────┤
│  [并排] [行内]  [忽略空白]  [显示字符]       搜索  自动换行  更多⌄        │
├──────────────────────────────────┬─────────────────────────────────────────┤
│  本机版本                         │  仓库版本                         ┊概览│
│  ~/.zshrc                         │  files/shell/zshrc                     │
│                                  │                                         │
│  18  export EDITOR="nvim"        │  18  export EDITOR="vim"               │
│  19  export LANG=zh_CN.UTF-8  →  │  19  export LANG=en_US.UTF-8            │
│                     [采用左侧 →] │                                         │
│  20  alias ll="ls -lah"          │  20  alias ll="ls -la"                 │
├──────────────────────────────────┴─────────────────────────────────────────┤
│ UTF-8 · LF · Shell · 42 行       [撤销] [保存到本机] [保存到仓库并收集]   │
└────────────────────────────────────────────────────────────────────────────┘
```

交互规则：

- 默认左侧为本机、右侧为仓库；标题始终标注真实路径，避免方向歧义。
- 默认只有选定的目标侧可编辑；切换编辑侧必须通过明确按钮。
- 差异块之间提供方向明确的“采用左侧 →”或“← 采用右侧”。
- 保存按钮明确写出目标，不使用单独的“保存”按钮同时改变两侧。
- 逐块采用只改变内存中的工作副本，点击保存后才写入磁盘。
- `Cmd/Ctrl+S` 保存当前编辑侧，`F7`/`Shift+F7` 浏览差异。
- 窗口较窄时自动使用行内模式，用户仍可手动切换。
- 可折叠相同段落，但保留上下文行数设置。
- 编辑器底栏显示编码、换行符、语言、光标位置、权限和文件大小。

若文件在打开后被其他程序修改，顶部显示持续可见的警告条，提供“重新加载并放弃编辑”“将外部版本作为新比较侧”和“另存为”。不提供直接强制覆盖按钮；高级菜单可在再次确认并创建额外备份后覆盖。

## 5. 配置详情抽屉

点击列表项后从右侧打开详情抽屉，不离开列表上下文：

- 名称与标签。
- 本机目标路径和仓库存储路径。
- 支持平台。
- 当前状态和最后收集时间。
- 文本文件的差异摘要和“比较并编辑”入口。
- “应用仓库版本”“收集本机版本”“编辑规则”。

二进制文件不显示文本差异，只显示大小、修改时间与摘要值。

## 6. 危险操作确认

确认窗口必须回答三个问题：

1. 将影响什么？
2. 原内容保存在哪里？
3. 出错后如何恢复？

“应用”确认页按新增、修改、跳过、冲突分类。主按钮使用动作名称，例如“备份并应用 4 项”，不使用含糊的“确定”。

```text
├──────────────────┴─────────────────────────────────────────────────────────┤

└────────────────────────────────────────────────────────────────────────────┘
```

- macOS 页面按 Formula、Cask、Tap、App Store 分组。
- 首次扫描结果只进入草稿；用户确认后才更新仓库清单。
- 安装计划页按阶段展示命令语义、权限、第三方来源和预计动作。
- 系统授权使用原生权限对话框，不在应用内收集 sudo 或 Apple ID 密码。
- 进度页逐包显示等待、下载、安装、成功、失败、跳过，可复制脱敏日志。
- App Store 未登录、`mas` 未安装、AUR 助手缺失等显示为可操作的前置条件。
- 默认主按钮是“安装缺失项”，删除和版本降级只放入高级流程。

## 8. 状态与反馈

- 短操作使用按钮内进度；Git 网络操作使用可取消的进度区。
- 成功提示短暂显示，同时写入活动记录。
- 部分失败必须展示逐项结果，不能把整批显示成笼统失败。
- Git 冲突进入专门的冲突页；首版只提供打开仓库目录和重试，不做不可靠的自动合并。
- 所有主要操作都可使用键盘完成，并提供可见焦点状态。

## 9. 视觉方向

- 遵循 macOS / Linux 桌面应用的紧凑信息密度。
- 使用中性灰背景与单一品牌强调色；状态不能只依赖颜色表达。
- 正文字号不低于 14px，主要点击区域不小于 36px。
- 默认跟随系统深浅色，避免自绘标题栏以保留平台习惯。
- 不引入大型组件库，建立少量基础组件以控制体积和一致性。
- 对比工作台允许更高信息密度，采用等宽字体、同步滚动和清晰的新增/删除/修改纹理；差异不能只用红绿色表达。
