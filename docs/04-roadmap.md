# 实施路线图

每个阶段都产出可运行、可验收的版本；上一阶段通过后再进入下一阶段。

## 阶段 0：设计确认（已完成）

交付：

- 产品范围与非目标。
- 主界面、首次引导和危险操作线框。
- 技术架构、仓库格式与安全模型。
- 分阶段验收标准。

退出条件：确认框架方向、MVP 范围、默认文件管理方式和对比编辑体验边界。

## 阶段 1：技术验证（已完成）

交付：

- Tauri 2 + Vue 3 最小桌面应用。
- macOS 与 Linux CI 构建。
- 原生文件/目录选择器。
- Rust 命令调用、异步进度事件和权限配置样例。
- Monaco Diff Editor 技术样例：并排/行内切换、可编辑工作副本、Worker 正常加载。
- Rust workspace、核心 crate、Tauri 适配层和前端 feature-slice 骨架。
- 自动生成 Rust DTO → TypeScript 类型的最小链路。
- 发布包体积、启动时间、空闲内存基线。

退出条件：两个平台均可启动，UI 不阻塞，权限没有开放通用 Shell/文件系统；核心 crate 不依赖 Tauri，模块依赖检查通过。

## 阶段 2：本地管理基础（已完成）

交付：

- 创建/打开仓库。
- 清单读取、校验、迁移与稳定写入。
- 添加文件和目录、状态扫描、文本差异。
- 内置配置知识库、已安装软件关联扫描和批量添加。
- 对比编辑工作台、逐块采用、差异导航、搜索和安全保存。
- 外部修改检测、编码/换行保持、大文件受限模式。
- 收集、dry-run、备份、应用、事务回滚。
- 配置列表和详情抽屉。

当前状态：单项文件流程、差异编辑、用户配置批量事务、故障回滚和重启后事务发现均已完成；目录合并策略、排除规则和逐文件删除预览仍需实现。

## 阶段 3：软件包迁移基础（进行中）

交付：

- 统一 `PackageProvider` 接口、软件包清单与状态比较。
- Arch Linux pacman 官方包扫描和恢复。
- Arch foreign/AUR 包识别，`paru`/`yay` 可选适配。
- Homebrew formula、cask、tap 的扫描和 Brewfile 恢复。
- Mac App Store 应用扫描、`mas` 前置检查和恢复。
- Flatpak 应用、安装范围和来源仓库的扫描与恢复。
- Linux Desktop Entry 扫描；AppImage/解压运行程序记录可执行路径、桌面入口和用户补充的下载来源。
- 软件包页面、安装计划、分组确认、进度和失败重试。
- Linux 系统授权适配、包管理器执行锁和脱敏日志。

当前状态：扫描、保存、缺失比较、单项安装，以及恢复向导中的阶段化缺失计划和顺序批量安装已实现；Flatpak 可按已记录的 origin 和用户/系统范围安装，Desktop Entry 可发现便携程序并保存其下载来源。安装动作会在执行前检查管理器、AUR helper 和提权工具，失败后可通过重新扫描从剩余项继续。便携程序目前保持人工下载与路径确认，仍缺少可信下载校验、持久任务进度、主动取消、硬件包角色隔离和干净系统验收。

## 阶段 4：Git 基础同步（进行中）

交付：

- Git 环境诊断。
- 克隆、fetch、提交、pull --rebase、push。
- SSH/HTTPS 认证错误的可操作提示。
- 分叉和冲突检测页。
- 操作日志脱敏。

当前状态：克隆、远程设置、提交、fetch、pull --rebase 和 push 已实现；冲突处理页、rebase 继续/中止和三方合并尚未完成。

## 阶段 5：完整桌面体验（进行中）

交付：

- 三步首次启动引导。
- 概览、同步、备份、设置页面。
- 软件包迁移加入首次恢复引导和概览状态。
- 批量操作与逐项结果。
- 深浅色、键盘操作、基础无障碍。
- 常见配置扫描建议和敏感文件警告。

当前状态：主要功能页面、新机预检、两阶段恢复向导、配置批量结果和异常退出后的事务处理入口已可用；软件包准备与配置恢复已合并展示并按阶段操作，但仍缺少跨阶段持久任务进度、主动取消、深色模式和完整无障碍验收。

## 阶段 6：发布准备（进行中）

交付：

- macOS App/DMG 与 Linux AppImage/Debian 包。
- macOS 签名/公证方案，Linux 包校验。
- 签名自动更新（可在首个公开版本后启用）。
- 用户文档、故障排查和卸载说明。
- 性能、安全和恢复演练报告。

当前状态：`0.1.0-alpha.1` 已建立统一版本、变更日志、安装校验文档和标签触发的草稿预发布流水线；流水线使用 Ubuntu 22.04 构建 Deb/AppImage，并分别使用原生 Apple Silicon 与 Intel runner 构建 DMG，汇总后生成 SHA-256 和公开仓库构建证明。稳定版标签会强制要求 Developer ID 签名与 Apple 公证凭据。Ubuntu/Arch 虚拟机已自动构建并启动验证 Linux 产物，本地 macOS DMG 也已验证。仍缺少真实 Developer ID 凭据验收、自动更新、RPM 和干净系统端到端迁移验收。

## 阶段 7：新机器恢复内核（进行中）

详细方案见 [新机器迁移就绪性审计与实现方案](05-migration-readiness.md)。该阶段将直接采用不兼容旧开发格式的 Manifest v2，优先实现机器预检、恢复计划、批量事务、平台适配器与受控权限代理。

当前状态：Manifest v2、机器事实预检、配置依赖排序、硬件/身份配置自动隔离、可解释恢复计划和桌面恢复向导已实现；用户级文件已支持不可变计划 ID、显式条目选择、写前逐项持久备份、事务记录、写后重新扫描校验和失败自动整批回滚。应用会发现异常退出遗留的事务，允许安全回滚或明确保留当前状态，未处理前禁止开始新恢复；恢复目标和备份元数据均执行 HOME、仓库重叠与符号链接逃逸校验。软件包缺失计划现已作为第一阶段与配置事务统一展示和顺序执行。下一步是把两个阶段提升为可取消、可续跑的单一持久任务，以及平台适配器与受控权限代理。

## 建议迭代顺序

| 里程碑 | 重点 | 可演示结果 |
|---|---|---|
| M0 | 设计 | 可评审文档与交互线框 |
| M1 | 框架验证 | 原生窗口、Rust 通信和编辑器级差异样例 |
| M2 | 文件闭环 | 离线对比编辑、管理和安全恢复配置 |
| M3 | 软件包闭环 | Arch/macOS 扫描和恢复缺失软件 |
| M4 | Git 闭环 | 两台设备模型之间同步 |
| M5 | 产品化 | 无终端完成首次使用 |
| M6 | 发布 | 可安装、可升级的软件包 |

## 首轮需要确认的产品决策

1. 默认采用“复制”模式。软链接作为高级选项延后；复制在跨设备和卸载后更可靠。
2. MVP 支持文件和目录，但不支持模板变量。
3. Git 冲突首版只检测和引导，不内置三方合并编辑器。
4. 不管理秘密；加密秘密仓库作为独立的后续能力。
5. 首个发布目标为 macOS arm64/x64 与主流 glibc Linux x64。
6. Monaco 对比编辑器属于 MVP；双向对比可编辑，Git 三方冲突合并器延后。
7. 使用编译期功能模块和 ports/adapters；动态第三方插件系统不属于 MVP。
8. 首批软件包 Provider 为 pacman、可选 paru/yay、Homebrew 和 mas；默认只恢复缺失项。
