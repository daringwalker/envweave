# 产品与功能设计

## 1. 产品目标

EnvWeave 用于在 Linux 与 macOS 之间迁移和维护用户环境。目标用户不需要理解软链接、Git 命令或目录布局，也能完成以下任务：

1. 把现有配置文件收集到本地仓库。
2. 用 Git 远程仓库在多台设备间同步。
3. 在新系统上预览差异后批量恢复配置。
4. 操作失误时找到自动备份并一键回滚。

## 2. 产品原则

- **默认不破坏**：覆盖前必须预览，原文件自动备份。
- **仓库可移植**：采用普通文件与 TOML；没有 EnvWeave 也能读懂和恢复。
- **秘密默认排除**：私钥、令牌、系统钥匙串等不主动收集。
- **本机优先**：离线可完成收集、应用、比较和回滚。
- **渐进式复杂度**：首次使用只显示必要选项，高级规则单独放置。
- **可解释**：每次操作都说明将读取、写入或删除哪些路径。
- **模块化演进**：文件、差异、备份、Git 和平台集成保持独立，可替换实现并复用于未来 CLI。

## 3. 第一版范围（MVP）

### 3.1 仓库

- 创建本地配置仓库。
- 打开已有本地仓库。
- 从 Git URL 克隆仓库。
- 配置远程地址和默认分支。
- 获取、提交、推送；发生冲突时停止并提示，不自动覆盖。

### 3.2 配置项

- 添加单个文件或目录。
- 为配置项设置名称、目标路径、支持平台和标签。
- 收集：本机路径 → 仓库。
- 应用：仓库 → 本机路径。
- 比较状态：一致、有改动、未安装、仓库缺失、平台不匹配。
- 批量选择、批量收集、批量应用。
- 删除配置项时分别提供“仅从清单移除”和“同时删除仓库副本”，不删除本机文件。

### 3.3 可视化对比编辑

可视化差异工作台是 MVP 核心功能，体验目标对齐 VS Code 与 Meld：

- 本机版本与仓库版本左右并排比较。
- 可切换单栏行内差异，窄窗口自动切换行内模式。
- 行级与字符级高亮，连接线与差异概览标尺。
- 上一个/下一个差异导航及差异总数。
- 逐块采用左侧或右侧内容，也可直接编辑工作副本。
- 搜索与替换、跳转到行、折叠未修改区域、多光标、撤销与重做。
- 语法高亮；首批覆盖 Shell、TOML、YAML、JSON、INI、Git config、Lua、Vim script 和纯文本。
- 显示/忽略空白差异、显示不可见字符、切换自动换行。
- 显示编码、换行格式、文件权限和只读状态。
- 编辑后保存到“本机”“仓库”或另存为；不会含糊地同时修改两侧。
- 离开未保存编辑器前必须提示保存、放弃或取消。
- 文件在编辑期间被外部修改时禁止静默覆盖，要求重新加载或重新比较。
- 二进制文件不进入文本编辑器，展示摘要、大小、修改时间和只读十六进制预览入口。

### 3.4 软件包迁移

软件包清单与 dotfiles 一同进入 Git 仓库，但由独立模块管理。首批支持：

- Arch Linux 官方仓库包（pacman）。
- Arch Linux foreign/AUR 包；自动检测 `paru`、`yay`，没有助手时提供人工处理清单。
- Homebrew formula、cask 和 tap。
- Mac App Store 应用，通过 `mas` 获取稳定的应用 ID。

主要能力：

- 扫描本机并按来源、类型、是否显式安装分组。
- 默认只选择用户显式安装/顶层软件，依赖由包管理器在恢复时解析。
- 支持搜索、批量选择、忽略规则和备注。
- 比较“期望清单”和本机状态：已安装、缺失、版本不同、来源不可用、不适用于本机。
- 生成分阶段安装计划，展示将调用的包管理器、包名、来源和是否需要提权。
- 默认只安装缺失项，不删除本机额外软件、不自动降级、不锁定历史版本。
- 安装可暂停、取消并逐项显示结果；失败项可单独重试。
- 采集时记录版本用于审计，恢复时默认安装软件源当前可用版本。
- 支持只导出命令或 Brewfile，允许高级用户在终端审阅执行。

Arch 恢复顺序为系统仓库包 → AUR 助手 → foreign/AUR 包。macOS 恢复顺序为 tap → formula → cask → App Store。AUR 构建脚本和第三方 tap 必须单独确认。

### 3.5 安全与恢复

- 应用前显示变更摘要。
- 覆盖前自动创建带时间戳的备份。
- 文件写入采用临时文件 + 原子替换。
- 展示最近操作记录和备份，支持恢复。
- 检测常见敏感文件名与疑似密钥内容，二次确认后才能加入。
- 任何 Git 冲突、权限错误、磁盘错误都不得留下半写入状态。

### 3.6 设置

- 浅色 / 深色 / 跟随系统。
- 界面语言（首版简体中文，结构预留国际化）。
- 仓库目录、备份目录、默认分支。
- Git 可用性、用户名、远程连接状态诊断。
- 包管理器可用性、版本、权限和账号状态诊断。

## 4. 延后功能

- 配置模板和变量替换。
- 针对不同主机的覆盖层。
- apt、dnf、Nix、Flatpak 和 Snap Provider。
- 加密秘密仓库。
- Windows 支持。
- 云服务账号体系。
- 内置语言服务器、格式化器和完整 IDE 补全。
- Git 三方冲突合并编辑器（双向对比编辑不延后）。

延后是为了先验证最重要的“收集—同步—恢复—回滚”闭环。

## 5. 核心用户流程

### 首次使用

1. 选择“新建仓库”“克隆远程仓库”或“打开本地仓库”。
2. 选择仓库路径；克隆时填写 Git URL。
3. 应用扫描建议，但不自动添加任何文件。
4. 勾选建议项并查看将保存的路径。
5. 完成收集；如果有远程仓库，再提交并推送。

### 新设备恢复

1. 克隆远程仓库。
2. EnvWeave 根据平台过滤不适用的项目。
3. 用户选择配置并查看差异和覆盖目标。
4. 先恢复软件包，再应用依赖这些软件的配置。
5. 应用配置；EnvWeave 自动备份已有文件。
6. 结果页列出成功、跳过、失败与备份位置。

### 软件包迁移

1. 扫描本机软件包并默认选择显式安装项。
2. 用户排除临时软件、驱动和不希望迁移的 AUR/tap 项。
3. 保存清单并通过 Git 同步。
4. 新设备读取清单，按当前平台匹配 Provider。
5. 用户审阅安装计划和第三方来源，授权后分阶段安装。
6. 安装完成后重新扫描，展示缺失项、失败原因和可重试操作。

### 日常同步

1. 首页显示本机配置与仓库的差异数量。
2. 用户进入配置列表，决定“收集本机改动”或“放弃本机改动并应用仓库版本”。
3. 对重要改动可先打开差异工作台，逐块采用或编辑后保存。
4. 同步页先拉取，再显示待提交变更，最后由用户确认推送。

## 6. 成功标准

- 新用户在 3 分钟内完成第一个配置文件的收集。
- 应用动作永远不会静默覆盖已有文件。
- 可从最近一次自动备份恢复到应用前状态。
- 仓库中的任意配置项都能从清单追踪到来源与目标。
- 对文本配置可以完成并排比较、逐块采用、编辑和安全保存，无需外部编辑器。
- 10 MB 以内文本文件打开和滚动流畅；超出阈值时给出性能提示并允许使用外部编辑器。
- Arch Linux 与 macOS 可以完成“扫描→保存清单→新机预览→安装缺失软件”的闭环。
- 软件包恢复不会默认删除额外软件、降级版本或静默运行 AUR/第三方 tap。
- 空闲状态不持续占用明显 CPU，应用安装体积保持合理。
